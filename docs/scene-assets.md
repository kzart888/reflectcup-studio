# Scene assets and provenance

The customer preview ships three versioned scene packages. The concept frames under `docs/assets/scenes/concepts/` were generated for this project on 2026-07-18 and are art-direction references only. They are ordinary perspective images, not PBR material maps, lightmaps or rotatable backgrounds.

## Selected directions

- `warm-craftsman-home`: the user selected **interior option 1**, a warm modern Craftsman breakfast corner. Reference: `warm-craftsman-home-selected.png`.
- `forest-camp-evening`: the user selected **outdoor option 2**, a dusk forest-camp table. Reference: `forest-camp-evening-selected.png`.
- `studio-neutral`: retained as the neutral optical diagnostic scene rather than a merchandising direction.

## Shipped runtime composition

### `warm-craftsman-home`

- Near/mid geometry: original React Three Fiber meshes authored for ReflectCup Studio: oak table, wall/window framing, bench, cushions and cabinet.
- Environment/background: [Warm Restaurant](https://polyhaven.com/a/warm_restaurant), Greg Zaal and Jarod Guest, Poly Haven CC0, 1K and 2K Radiance HDR.
- Table material: [Fine Grained Wood](https://polyhaven.com/a/fine_grained_wood), Rob Tuytel, Poly Haven CC0, 1K colour/OpenGL normal/roughness JPEGs.
- Shadow: `v2/table-shadow.png`, generated locally and deterministically by `pnpm scene:bake`.

### `forest-camp-evening`

- Near/mid geometry: original React Three Fiber meshes authored for ReflectCup Studio: walnut table, ground, instanced trunks, tent, camp chair and lantern. Object placement is invariant across quality tiers.
- Environment/background: [Dark Autumn Forest](https://polyhaven.com/a/dark_autumn_forest), Dimitrios Savva and Greg Zaal, Poly Haven CC0, 1K and 2K Radiance HDR.
- Table material: [Dark Wood](https://polyhaven.com/a/dark_wood), Dario Barresi, Dimitrios Savva and Rico Cilliers, Poly Haven CC0, 1K colour/OpenGL normal/roughness JPEGs.
- Trunk material: [Bark Brown 01](https://polyhaven.com/a/bark_brown_01), Rob Tuytel, Poly Haven CC0, 1K colour/OpenGL normal/roughness JPEGs.
- Shadow: `v2/table-shadow.png`, generated locally and deterministically by `pnpm scene:bake`.

### `studio-neutral`

- Environment: [Studio Small 08](https://polyhaven.com/a/studio_small_08), Sergej Majboroda, Poly Haven CC0, 1K Radiance HDR. Its adjacent `ASSET.md` records the downloaded source MD5.
- Near geometry: original neutral studio/table geometry.
- Shadow: `v2/table-shadow.png`, generated by the same local bake task. The original root HDR is retained for the earlier studio release.

All three scenes share the content-addressed `public/scenes/shared/f30bf914fdcc7fc6/cup-contact-ao.png`. The bake tool creates 1024 × 1024 RGBA layers from deterministic analytic soft ellipses with Sharp. These files contain no customer data or third-party imagery. They remove realtime shadow-map stair stepping at very low GPU cost, but are artistic decals rather than Cycles-baked geometry-aware AO or illumination.

Poly Haven distributes these assets under [CC0](https://polyhaven.com/license/). The public filenames are project-local renames of exact upstream 1K/2K files; their official byte sizes and MD5 values match. This provenance must remain attached if the assets are later moved or recompressed.

The byte sizes and SHA-256 checksums of the exact files in this repository are recorded in [`scene-asset-manifest.md`](scene-asset-manifest.md) and mirrored by the shared, client-safe release manifest. The immutable scene checksum stored with a confirmed design covers those files plus the visible scene parameters and renderer/geometry pipeline versions.

## Quality catalog

`src/scenes/release-manifest.ts` declares the content hashes and low, medium and high tier composition; `src/scenes/catalog.ts` derives the runtime URLs and exact download bytes from it. Unit tests hash the files, compare their exact sizes and enforce 4/7/12 MB ceilings. Mobile/save-data devices use the low tier. Desktop starts at medium, promotes to the 2K high tier while idle and falls back to low when the performance monitor declines. Tier changes preserve the visible object layout. Loader sources are retained for the current and previous scene; older non-shared entries are cleared after a successful switch.

The release checksum uses canonical JSON and SHA-256 over the scene ID/version, asset manifest, tier composition, visible parameters and explicit rendering-pipeline versions. Server and browser catalogs consume the same pure-data manifest, so confirmation provenance cannot drift between them. All current releases are v2: the v1 identity-only checksums had already been published and remain pinned for old snapshots rather than being rewritten. SceneBackdrop and the reflective subject obtain every asset path from the selected descriptor; v2 assets use immutable `/v2/` paths (the shared AO uses its hash prefix). Published releases are immutable: change any visible input or rendering implementation only by publishing a new version and checksum.

The historical v1 records did not bind deployable scene assets, so they support snapshot provenance/audit only, not byte-identical visual replay. Draft sessions select the current published release by scene ID; a confirmed v1 snapshot keeps its original version/checksum in the immutable design document. Production PNGs are unaffected because scene identity never enters the optical renderer.

## Pipeline limitation and next asset pass

Blender and `toktx`/glTF compression tooling were unavailable in the implementation environment. Therefore the current scene packages do **not** yet contain:

- Cycles-baked direct/indirect illumination, geometry AO or UV2 lightmaps;
- authored GLB near/mid meshes, Meshopt compression or geometry quantization;
- ETC1S/UASTC KTX2 texture sets;
- a far 360° panorama and unclipped reflection HDR rendered from the same authored scene;
- validated low/medium/high visual parity on mobile GPUs.

The shipped HDR backgrounds and procedural geometry are an efficient first visual pass. They can have small spatial or lighting mismatches and must not be described as the final same-scene photoreal bake. The next asset pass should reconstruct the selected concepts in Blender around the cup optical origin, bake the static layers, export matching panorama/HDR orientation, transcode, and then add a content-addressed scene manifest.
